# 产品文档

## 产品定位

`Fortress War` 是一个手机竖屏的轻策略攻防原型。玩家通过最少的点按操作，在一张可快速进入战局的预设战略地图上占城、运兵、扩张，并与多个独立电脑势力竞争最后统治权。

## 目标用户

- 喜欢轻量即时策略、但不希望被复杂资源系统拖慢节奏的玩家
- 主要在手机上体验，希望单手触控即可完成核心操作的玩家
- 需要快速验证规则闭环与战斗反馈的原型测试用户

## 核心体验

- 进入对局后，快速理解“城市会产兵、道路决定进攻、路上部队会遭遇战”这条主循环
- 玩家通过两次点击直接创建或关闭持续出兵路线，尽量不被额外弹窗打断
- 多个电脑势力彼此攻伐，地图战线会持续变化，不是单纯的玩家打一个固定敌人
- 战场支持拖拽与缩放浏览，让竖屏也能容纳更有纵深的多线战局
- UI 维持竖屏、低认知负担、高可读性，暂停和说明不应打断主战场观察

## 当前功能清单

- 默认使用中国风抽象预设地图开局，并按 `2` 至 `5` 方切换出生点配置
- 仓库保留随机地图生成器，作为后续可并存的另一种地图来源
- 地图尺寸大于单屏，支持鼠标 / 单指拖拽与滚轮 / 双指缩放浏览
- 城市使用古代中国风格命名与符号化 marker 展示
- 玩家 + 多个独立电脑势力 + 中立城市的多方混战
- 城市拥有防御、产能和等级属性，其中等级决定人口上限
- 占领城市会按各自产能自动产兵，中立城市不产兵
- 选中己方城市后可直接消耗驻军升级等级、防御或产能
- 玩家可跨图运兵；进攻必须沿道路相邻
- 玩家和 AI 共用持续出兵任务系统；`A -> B` 路线会在源城新增士兵时自动派出 10 人
- 同一源城存在多条持续路线时，会按轮转顺序自动调度
- 路线不会因目标换手自动消失，而是会动态切换为进攻或运兵；源城失守则立刻清空该城发出的全部路线
- 行军单位具象化展示，行军时间与距离相关
- 路上不同势力部队相遇会先战斗抵消
- 到城后再结算攻城或增援
- 电脑势力支持总方数、难度、风格配置
- 背景音乐与操作音效
- 暂停、重新开始、开始说明、结束提示

## 交互原则

- 优先保证单指点击和鼠标左键一致
- 操作反馈要及时，尽量通过顶部状态栏与音效提示玩家发生了什么
- 暂停应尽量轻量，不遮挡战场观察
- 城市图标和信息布局要在小尺寸下仍能快速识别

## 非目标

- 当前不引入资源采集、科技树、技能系统、兵种克制
- 当前不做联网、排行榜、存档
- 当前不做复杂经济系统和多段路径规划
- 当前不把一次性出兵弹窗作为主流程；相关接口仅保留给后续扩展
