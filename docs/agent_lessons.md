# Agent Lessons

## 2026-03-22

- 不要在同一轮实现里同时保留两套核心模型并混接主场景。此次先后出现了 `City/GameState/Faction` 与另一套原型命名结构，导致场景脚本引用和类型解析互相打架。
- 手写 `.tscn` 前必须先读取仓库中的真实场景结构。此次一开始假定主场景只有 `MapView/Hud`，实际仓库已有 `Cities/UILayer/CityTemplate` 结构，造成节点路径错误。
- Godot 原型阶段优先使用 `preload` 和更稳妥的弱类型跨脚本引用，先保证装配稳定，再逐步加强静态类型。
- 设计 AI 难度和风格时，要把“是否出手”和“偏好打谁”拆成两个维度。此次一开始把保守阈值直接压在最终派兵数上，导致简单防御型在部分局面下完全不行动；修正方式是用源城总体优势决定能否出手，再用评分函数决定更偏向玩家城还是中立城。
- 引入第三方字体或素材前，先验证下载地址和许可证文件路径，再写自动化命令。此次先假设 `notofonts/noto-cjk` 根目录直接提供许可证文本，结果命中 404，虽然字体已下成功，但补许可证多走了一步。
- 新增城市属性或经营操作时，要同步校准旧测试基线。此次引入“防御计入攻城门槛”和“升级消耗驻军”后，若直接沿用旧断言，会把规则 bug 和测试预期漂移混在一起。
