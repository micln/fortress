# 2026-04-05 Agent Notes

## 错误复盘 1

- 任务背景：修复“`a -> b -> c` 中继链在 `b` 满员时会吞兵”的持续出兵逻辑。
- 错误现象：把规则错误实现成“只有 `overflow_count` 才继续触发 `b -> c` 调度”，遗漏了“运入本身也是新增士兵事件”。
- 根因分析：把“结果状态（溢出多少）”误当成“事件语义（新增几名士兵）”；先整包并城再补派，丢失了逐兵到达的时序。
- 正确做法：友军运兵到达同阵营城市时，要逐兵结算；每 1 名到达士兵都立即触发 1 次持续调度。这里的“新增士兵”既包括本地产兵，也包括外部运入。
- 防再犯措施：
  - 以后遇到“事件驱动”规则时，先写出事件源和触发次数，再决定状态字段，不再直接从最终聚合结果反推事件次数。
  - 先补一条能表达时序的回归测试，再改实现，避免再次把逐步过程做成事后补偿。
