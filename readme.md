# Fortress War

一个使用 Godot 4.6 开发的竖屏城堡攻防原型项目，目标平台为手机，交互以触控为主，同时兼容鼠标。

## 当前能力

- 当前默认使用一张中国风抽象预设地图开局，城市与道路布局固定为中型战略图
- 预设地图仍支持 `2` 至 `5` 方开局，不同总方数会切换不同出生点配置
- 仓库仍保留随机地图生成器，作为后续可并存的另一种地图来源
- 城市采用更偏古代中国城池的符号化图标，而不是简单圆点
- 每座城市都有防御、产能和等级三项属性：防御越高越难被攻下，产能越高产兵越快，等级越高对应更高的人数上限
- 产能升级现在每次会带来更明显的跳升，便于中后期通过经营拉开持续出兵强度差距
- 被占领城市会按各自产能持续产兵，达到人数上限后停止产兵，中立城市不产兵
- 选中己方城市后，可直接消耗城内士兵升级等级、防御或产能
- 玩家和多个独立电脑势力都通过“持续出兵任务”从一个城市向目标城市持续派兵
- 己方城市到己方城市会执行运兵，且可跨越整张地图直接输送，不受道路相邻限制
- 玩家默认只使用持续出兵：先点己方城市 A，再点目标城市 B，就会直接创建或关闭 `A -> B` 路线
- 同一个源城存在多条持续路线时，会在每次产兵后按轮转顺序自动派出 10 人
- 城市每新增 1 名士兵都会触发一次持续调度；这里的“新增”既包括本地产兵，也包括其他城市运送进来的援军
- 持续路线不会因为目标换手而自动消失，而是会根据目标当前归属动态变成进攻或运兵
- 若源城失守，会立刻取消该城发出的全部持续路线；后续即使夺回也不会自动恢复
- 对局中可随时手动暂停，再继续当前战局，暂停不会弹出遮挡战场的大面板
- 开局可选择总方数，默认 5 方，即 1 个玩家加 4 个独立电脑势力
- 电脑 AI 提供难度和风格选项，默认是简单难度、防御型风格；AI 现在和玩家共用同一套持续出兵任务系统
- 进攻到达时，攻方人数需要大于等于“守军 + 城市防御”才会占领目标城市
- 出兵不会瞬间完成，地图上会出现实际行军单位，行军时间与城市距离相关
- 不同势力的行军单位若在道路上相遇，会先战斗抵消，剩余部队再继续前进
- 战斗会在行军单位抵达目标城市时才结算
- 同一帧多支队伍同时到达时，先结算进攻，再结算运兵
- 提供程序生成的背景音乐与操作音效
- 提供四层分层的程序背景：主草地底层、不规则土地色块、分段草纹和极轻氛围块；背景只在主场景 `_draw()` 中绘制，不新增独立背景节点，且始终位于道路、城市和行军单位之下
- 城市 marker 默认使用短文本（`城/关/枢/腹`）；emoji 只在显式开关打开时启用，避免默认字体链路的缺字风险。可通过 `ProjectSettings` 键 `fortress_war/ui/use_marker_emoji` 或环境变量 `FORTRESS_WAR_USE_MARKER_EMOJI=1` 显式开启
- 地图世界尺寸按移动端/桌面端采用不同策略：手机会拉开城池间距，桌面会进一步收紧默认战场，并让开局镜头轻微偏向玩家出生城，避免首屏被顶部 HUD 挡住
- 地图世界尺寸大于单屏，可拖拽浏览更远的战线与城市，并支持缩放查看
- 提供一个最小可玩的竖屏原型与核心规则测试

## 目录结构

- `scenes/`：场景文件
- `scripts/domain/`：领域规则与实体
- `scripts/application/`：流程编排、地图来源、AI 决策
- `scripts/presentation/`：界面渲染、输入交互、HUD
- `tests/`：核心规则测试
- `docs/`：设计与约束文档

## 代码入口与结构

- 主场景：`scenes/main/main.tscn`
- 主控制器入口壳：`scenes/main/main.gd`
  - 该文件仅负责 `extends` 到分层脚本，保持场景引用路径稳定
- 主控制器分层（presentation 内部分层）：
  - `scenes/main/main_bootstrap.gd`：装配/初始化（_ready 接线）
  - `scenes/main/main_flow.gd`：主流程与 UI 编排
  - `scenes/main/main_context.gd`：共享基础与工具
  - `main_input_handler.gd`：输入/手势状态机与“空白释放取消选城”语义

## 运行方式

1. 使用 Godot 4.6 打开项目根目录
2. 运行主场景 `scenes/main/main.tscn`
3. 阅读开局说明后点击“开始游戏”
4. 点击蓝色己方城市，再点击目标城市，直接创建或关闭持续出兵路线
5. 点击敌方或中立城市时，只有相邻且有道路连接才可进攻
6. 点击己方城市时，无论是否相邻都可直接运兵
7. 城市会分层显示名称、当前人数上限、防御、产能和等级，例如 `长安`、`18/35`、`Lv.2 防2 产1.1`
8. 开始游戏前可在说明面板中选择总方数、难度与风格，默认是 `5 方 | 简单 | 防御型`
9. 当前第一阶段会直接进入一张中国风抽象预设地图，不再默认生成随机地图
10. 当前默认不再走出兵数量弹窗；一次性出兵接口仅预留扩展，尚未接入主流程
11. 右侧 HUD 会显示所有进行中的持续路线，便于观察玩家与电脑 AI 的任务状态
12. 对局中可用底部“暂停”按钮冻结当前战局，再点击继续恢复
13. 选中后可点击空白区域或底部“取消选择”按钮立即取消；升级按钮会直接显示在被选城市旁边
14. 桌面端可按住鼠标左键拖拽地图；移动端和 Web 可单指拖动战场查看更远区域
15. 桌面端可用鼠标滚轮缩放地图；触屏设备可双指缩放地图
16. 想重新来一局时，使用底部“重新开始”或结束面板按钮

## 分辨率与适配

- 项目不再绑定固定逻辑分辨率。
- 桌面端（非 Web、非移动平台）启动时默认进入窗口最大化，减少窗口尺寸对首屏体验和战场可视面积的影响。
- 启动时会读取当前设备/窗口实际尺寸，并将其作为内容分辨率。
- 窗口尺寸变化（桌面拖拽、移动端旋转、Web 画布变化）时会实时同步内容分辨率，确保画面持续自适应。
- 当窗口放大时，地图世界尺寸会自动扩展，确保战场背景持续覆盖可视区，避免出现黑边。

## 测试方式

可通过 Godot headless 模式执行 `tests/test_runner.gd` 来验证当前原型核心规则：

```bash
HOME=/tmp XDG_DATA_HOME=/tmp godot --headless --path . -s res://tests/test_runner.gd
```

本地 Web 冒烟检查可额外执行：

```bash
scripts/tools/web_smoke_check.sh
```

手工验收重点：

- 背景四层是否都压在道路、城市和行军单位之下
- 装饰层是否保持低对比、单层 alpha 不超过 `0.22`
- 画面里是否存在会被误认成可交互节点或障碍的高对比封闭图形
- 城市 marker 是否默认显示短文本，且只有在显式开关打开时才切到 emoji
- 若要联调 emoji 路径，是否已设置 `fortress_war/ui/use_marker_emoji` 或 `FORTRESS_WAR_USE_MARKER_EMOJI=1`

## Web 试玩版

- 仓库在 `master` 分支收到推送后，会通过 GitHub Actions 自动构建 Web 试玩版并发布到 GitHub Pages。
- 发布工作流定义在 `.github/workflows/deploy-web.yml`，使用 Godot `4.6.1` 和 `export_presets.cfg` 中的 `Web` preset。
- 首次启用时，需要在 GitHub 仓库设置中把 Pages 的发布来源切换为 GitHub Actions。
- 工作流会在导出后校验 `index.html`、`index.js`、`index.wasm`、`index.pck` 和 `404.html` 是否齐全，避免部署空包或残缺包。
- 本地手机联调不要再使用 `python3 -m http.server` 的纯 HTTP 方式。Godot Web 在当前导出配置下要求 Secure Context，本地请使用 `scripts/tools/serve_web_https.sh` 启动 HTTPS 服务。

## Web 中文显示说明

- Web 导出不能假设浏览器宿主环境一定提供可用的中文字体。
- 项目显式携带 `assets/fonts/NotoSansSC-Regular.otf` 作为默认 UI 字体，并用于场景绘制路径中的文本渲染，避免按钮、城市名和说明文案在 Web 版中出现方块字或缺字。
- 城市 marker 同样默认走短文本，emoji 仅作为显式开关路径，不作为默认展示方案；显式开关支持 `fortress_war/ui/use_marker_emoji` 与 `FORTRESS_WAR_USE_MARKER_EMOJI=1`。
